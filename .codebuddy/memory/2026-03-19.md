## 添加 Vercel 部署和混合渲染配置

- 在 `nuxt.config.ts` 中添加了 `routeRules` 混合渲染配置：
  - `/`: prerender（预渲染静态页面）
  - `/blog/**`: ISR 60 秒（增量静态再生成）
- 说明：~~Nuxt Content v3 生产环境使用 sql.js (WASM) 而非 better-sqlite3~~（这是错误的分析）
- **修正**: Nuxt Content v3 在 Vercel 上**仍然使用 `better-sqlite3` native 模块**，数据库文件存在 `/tmp/contents.sqlite`
- 数据（sql_dump.txt）通过 `database.compressed.mjs` 打包在云函数代码内，不依赖 CDN
- Vercel 会自动检测 Nuxt 项目并使用 `vercel` preset，无需显式配置

## 修复 Vercel 部署 /blog 路由 404 问题

- **问题**: 部署到 Vercel 后 `/blog` 返回 Vercel 平台级 404（NOT_FOUND），而非 Nuxt 应用级错误
- **根因**: `routeRules` 中 `/blog/**` 使用的是通配符，只匹配 `/blog/xxx` 子路径，不包含 `/blog` 本身。导致 Vercel 构建时只生成了 `blog/[...]-isr.func`（处理子路由），没有为 `/blog` 列表页生成对应的 serverless function（`blog-isr.func`）
- **修复**: 在 routeRules 中单独添加 `'/blog': { isr: 60 }` 规则
- **验证**: 重新 `NITRO_PRESET=vercel nuxt build` 后，`.vercel/output/functions/` 目录下新增了 `blog-isr.func`，`config.json` 中也新增了 `/blog` → `/blog-isr` 的路由规则

## 排查 Vercel 上 content 查询 500 错误（所有文章页）

- **现象**: `/blog/getting-started` 等文章页在 Vercel 上显示 "404 - 文章未找到"，实际是 SSR 渲染时 `queryCollection` 返回 500 Server Error
- **验证**: `curl -X POST /__nuxt_content/content/query` 返回 500 Server Error
- **根因分析**:
  - Nuxt Content v3 Vercel preset 使用 `better-sqlite3` (native) 连接 `/tmp/contents.sqlite`
  - 数据通过 `database.compressed.mjs`（base64 gzip 压缩的 SQL dump）打包在云函数内
  - 运行时通过 handler 从 mjs 文件读取 → 解压 → 导入到 SQLite
  - **但 `better-sqlite3` 的 native .node 模块可能在 Vercel Lambda 环境中编译/加载失败**
  - 需要 `package.json` 中添加 `"onlyBuiltDependencies": ["better-sqlite3"]` 确保 CI 上正确编译
- **修复**: 在 `package.json` 添加了 `onlyBuiltDependencies` 配置
- **待验证**: 需要 push 到 GitHub 触发 Vercel 重新部署后验证
